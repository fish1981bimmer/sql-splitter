# SQL Splitter 安全说明

## 安全措施

### 1. 数据序列化安全

**问题：** 早期版本使用 pickle 进行数据序列化，存在反序列化漏洞风险。

**修复：** v2.1+ 版本已将 pickle 替换为 JSON，确保数据序列化安全。

- 检查点文件使用 JSON 格式存储
- 数据验证：加载时验证 input_file 字段匹配
- 使用 dataclass 的 to_dict() 和 from_dict() 方法进行类型转换

### 2. 依赖管理

**当前状态：** 仅使用 Python 标准库，无第三方依赖。

**优势：**
- 减少供应链攻击风险
- 无需额外安装依赖
- 兼容性更好

**依赖列表：** 见 requirements.txt

### 3. 内存/上下文污染防护

**措施：**
- 所有全局变量均为不可变常量（frozenset）
- 无可变全局状态
- 使用函数参数传递状态
- 避免全局变量修改

### 4. 文件操作安全

**措施：**
- 使用 pathlib 进行路径操作，避免路径遍历攻击
- 检查文件存在性后再操作
- 使用 try-except 处理异常
- 限制检查点文件目录

### 5. 输入验证

**措施：**
- 验证输入文件路径
- 验证输出目录路径
- 验证 SQL 方言参数
- 检查点数据验证

## 安全最佳实践

1. **不要从不可信来源加载检查点文件**
   - 检查点文件存储在用户目录下
   - 加载时验证 input_file 字段

2. **定期清理旧检查点**
   - 使用 clear_old_checkpoints() 方法
   - 默认保留 7 天

3. **审查代码**
   - 所有代码已通过安全扫描
   - 无已知漏洞

4. **版本控制**
   - 使用 Git 进行版本管理
   - 提交历史可追溯
